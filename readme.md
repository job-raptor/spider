# 技术方案

后台系统基本可分为三大部分:爬虫，数据，查询

## 爬虫

#### 爬虫技术

- scrapy较好,但可能的问题是太重
- requests较轻,几行代码就可以开工,但容易陷入重复前人工作,代码乱的境地

#### 反反爬

- UA,IP等伪造
- cookie处理
- 验证码破解
- 时间策略破解

#### 更新机制和抓取策略

职位信息肯定是越新越好,同时之后的分析中需要分析的是即时职位情况和历史变化情况两种

- 暂定一天一抓,抓取极限为?
- 检查页面是否已抓的方式为?

## 数据

#### 存储

- 存储方案量小时mysql为好,对比nosql方便进行结构化查询,处理和更新
- 数据量中等时,讨论较多的方案是mongodb
